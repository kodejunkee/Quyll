Quyll UI/UX Bible v1.0

---

Design Philosophy

Quyll is a professional desktop writing application.

The interface should feel:

- Clean
- Modern
- Elegant
- Fast
- Calm
- Focused

The application should never feel crowded or overwhelming.

Writing always comes first.

Everything else supports writing.

---

Design Principles

1. Focus First

The writing editor is always the center of attention.

Panels and tools should stay out of the way until needed.

---

2. Progressive Disclosure

Do not show every option immediately.

Only reveal additional tools when the writer requests them.

---

3. Consistency

Every page should follow the same spacing, typography, colors and interaction patterns.

The user should never wonder how something works.

---

4. Smooth Navigation

Moving around the application should feel instant.

No unnecessary loading screens.

---

5. Calm Interface

Avoid excessive animations.

Animations should communicate state changes rather than decorate the interface.

---

Window Layout

The desktop application consists of four primary regions.

Left Sidebar

Contains navigation.

Examples

- Dashboard
- Chapters
- Characters
- Locations
- Organizations
- Species
- Items
- Magic Systems
- Lore
- Timeline
- Plot Planner
- Settings

Sidebar should support collapsing.

---

Main Workspace

Displays the current page.

This is where users perform their primary task.

---

Inspector Panel

Appears on the right.

Displays contextual information.

Examples

- Character details
- Location details
- Timeline entry
- AI panel (future)

Can be collapsed.

---

Floating Reference Bubbles

Optional.

Users may pin references.

Pinned references appear as floating circular buttons.

Clicking opens the associated entity.

---

Home Screen

Displays

- Recent Projects
- Create Project
- Open Project
- Search Projects

Minimal layout.

No unnecessary information.

---

Dashboard

Displays project overview.

Examples

- Total Chapters
- Total Characters
- Total Locations
- Story Progress
- Recent Activity

---

Chapter Editor

The editor is Quyll's most important screen.

Features

- Large writing area
- Distraction-free
- Rich Text Editing
- Autosave
- Word Count
- Reading Time
- Current Chapter

No unnecessary side panels unless requested.

---

Entity Pages

Every entity page follows the same layout.

Header

- Name
- Type
- Image

Body

Organized into cards.

Examples

- General Information
- Description
- Notes
- Relationships

---

Cards

Cards should have

- Rounded corners
- Soft shadows
- Comfortable spacing

Avoid excessive borders.

---

Typography

Priorities

- Readability
- Comfortable spacing
- Clear hierarchy

Use three heading sizes only.

Avoid unnecessary font variations.

---

Color Palette

Dark Theme

Primary background

Near black.

Secondary surfaces

Dark gray.

Accent

Single accent color throughout the application.

Avoid rainbow interfaces.

Light Theme

Neutral whites and light grays.

Maintain identical layout.

---

Icons

Use a single icon family throughout the application.

Icons should always support labels.

Avoid icon-only navigation where possible.

---

Buttons

Primary

Used for important actions.

Examples

- Save
- Create
- Continue

Secondary

Less important actions.

Examples

- Cancel
- Close

Danger

Delete actions only.

---

Forms

Every form should include

- Labels
- Placeholder text
- Validation
- Helpful error messages

Never rely only on placeholders.

---

Hover Cards

Hover Cards should appear quickly.

Display only essential information.

Do not overwhelm the user.

Include an "Open" action for full details.

---

Keyword Appearance

Keywords should be visually distinct without looking like hyperlinks.

Use subtle highlighting.

Writers should immediately recognize linked entities.

---

Search

Global Search should be accessible from anywhere.

Results grouped by category.

Examples

Characters

Locations

Lore

Timeline

Chapters

---

Empty States

Every empty page should encourage action.

Example

"No characters yet."

Button

Create Character
Avoid blank screens.

---

Loading States

Use skeleton loaders instead of spinners whenever practical.

Loading should feel responsive.

---

Animations

Animations should be

- Short
- Smooth
- Purposeful

Avoid flashy effects.

---

Keyboard Shortcuts

Support keyboard-first workflows.

Examples

- New Chapter
- Save
- Search
- Toggle Sidebar
- Toggle Inspector
- Focus Editor

---

Accessibility

Support

- Keyboard navigation
- High contrast mode
- Adjustable font sizes
- Screen scaling

Performance Goals

UI should feel

- Instant
- Responsive
- Smooth

Large projects should remain usable.

---

Overall Experience

When someone opens Quyll for the first time, they should immediately feel that it is a professional writing application rather than an AI tool.

The interface should inspire creativity, reduce friction, and help writers stay immersed in their stories.
